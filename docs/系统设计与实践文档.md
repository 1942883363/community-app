## 1. 项目概述

社区便民资讯平台是一个面向社区居民的综合信息化服务平台，包含**管理后台**（Web）和**用户小程序**两个前端，通过 **Python FastAPI 后端** 统一提供数据服务。

**核心功能模块**：

| 模块 | 管理后台 | 小程序端 |
|------|---------|---------|
| 资讯管理 | 发布/编辑/置顶/分类 | 浏览/点赞 |
| 便民电话 | 分类+条目管理 | 分类浏览/拨号 |
| 社区活动 | 发布/管理/报名统计 | 浏览/报名 |
| 周边商家 | 分类+商家管理 | 分类浏览/详情 |
| 意见反馈 | 查看/处理 | 提交反馈 |
| 公交查询 | — | 线路搜索/实时到站 |
| 用户管理 | 列表/编辑/删除/改密 | 注册/登录/个人资料 |
| 图片审核 | 审核/通过/拒绝 | — |
| 数据仪表盘 | 六维度统计卡片 | — |

### 5.2 统一响应格式

所有 API 返回统一的 JSON 结构：

```json
// 成功
{ "code": 200, "message": "success", "data": { ... } }

// 错误
{ "code": 404, "message": "该手机号未注册", "data": null }

// 分页
{
  "code": 200,
  "message": "success",
  "data": { "items": [...], "total": 100, "page": 1, "page_size": 20 }
}
```

三要素：`code`（状态码）、`message`（描述）、`data`（实际数据）。

> **注意**：即使业务错误（如 404）也返回 HTTP 200，错误码放在 `code` 字段中。这是当前项目的设计约定。

实现位于 [utils/response.py](file:///d:/实习项目/backend/utils/response.py)，提供三个辅助函数：
- `success_response(data, message)` — 构造成功响应
- `error_response(code, message)` — 构造错误响应
- `paginated_response(items, total, page, page_size)` — 构造分页响应

### 5.3 认证与鉴权体系

系统有两套独立的鉴权机制：

#### 5.3.1 管理后台鉴权（JWT）

```
POST /api/auth/login {username, password}
  → bcrypt 验证密码
  → 签发 JWT Token (HS256, 8h 过期)
  → 返回 {access_token, token_type: "bearer"}

后续请求: Authorization: Bearer <token>
  → JWT 解码 → 查找 admin_users 表 → 返回 AdminUser 对象
```

实现细节：
- 密码哈希：`bcrypt.hashpw(password + gensalt())` 存储到 `password_hash`
- JWT 签发：payload 包含 `{user_id, username, exp}`，密钥从 `.env` 读取
- Token 验证：`get_current_user` 依赖注入，401 时抛出 HTTPException
- Token 过期后不会自动续期，需要重新登录

#### 5.3.2 小程序端鉴权（openid）

```
小程序首次启动:
  wx.getStorageSync('openid') → 无 → 生成 UUID → 存储

后续请求:
  X-User-Id: <uuid> → 后端按 openid 查找/创建用户
```

### 5.4 用户系统（routers/user.py）

#### 5.4.1 核心设计

用户身份通过 `openid`（设备唯一标识）绑定。关键接口：

| 端点 | 方法 | 鉴权 | 功能 |
|------|------|------|------|
| `/api/users/register` | PUT | X-User-Id | 注册新用户（昵称+手机号+密码） |
| `/api/users/login` | POST | X-User-Id | 手机号+密码登录 |
| `/api/users/logout` | POST | X-User-Id | 退出登录（清空 openid） |
| `/api/users/me` | GET | X-User-Id | 获取当前用户信息 |
| `/api/users/me` | PUT | X-User-Id | 修改个人资料 |
| `/api/users` | GET | JWT(admin) | 后台用户列表（分页） |
| `/api/users/{id}` | GET/PUT/DELETE | JWT(admin) | 后台用户 CRUD |

#### 5.4.2 注册流程（详细实现）

```python
@router.put("/register")
def register_user(data: UserRegister, x_user_id: str = Header(...), db: Session = Depends(...)):
    # Step 1: 校验输入
    #   - 昵称非空
    #   - 手机号 11 位
    #   - 密码 >= 6 位

    # Step 2: 唯一性检查
    #   - 昵称唯一（409）
    #   - 手机号唯一（409）

    # Step 3: 解绑当前设备旧账号
    #   - 如果当前 openid 已有注册用户 → 设 openid = ""
    #   - db.flush() 确保 UNIQUE 约束不冲突

    # Step 4: 创建新用户
    #   - User(openid=x_user_id, nickname=..., phone=...)
    #   - password_hash = pwd_context.hash(password)  # bcrypt
    #   - 可选 avatar

    # Step 5: 头像审核
    #   - 有头像 → create_image_review("user_avatar", auto_approve=True)

    return success_response(data, "注册成功")
```

> **设计要点**：同一台设备注册新账号时，旧用户的 `openid` 会被清空（设为空字符串）以解除绑定，而非删除旧数据。旧用户仍然可以通过手机号+密码登录。

#### 5.4.3 登录流程（详细实现）

```python
@router.post("/login")
def login_user(data: UserLogin, x_user_id: str = Header(...), db: Session = Depends(...)):
    # Step 1: 按手机号查找用户
    target = db.execute(select(User).where(User.phone == phone)).scalar_one_or_none()
    if target is None:
        return error_response(404, "该手机号未注册，请先注册")

    # Step 2: bcrypt 验证密码
    if not target.password_hash:
        return error_response(400, "密码错误")
    if not pwd_context.verify(password, target.password_hash):
        return error_response(400, "密码错误")

    # Step 3: openid 绑定切换
    current = db.execute(select(User).where(User.openid == x_user_id)).scalar_one_or_none()
    if current and current.id != target.id:
        current.openid = ""  # 解绑旧账号
        db.flush()
    target.openid = x_user_id  # 绑定目标账号

    return success_response(data, "登录成功")
```

> **设计要点**：用户可以在任意设备上登录自己的账号。登录操作实质是"将目标用户的 `openid` 设置为当前设备的 openid"。当前设备上已有的旧账号 bindings 会被自动解除。

#### 5.4.4 GET /me 设计

```python
@router.get("/me")
def get_self(x_user_id: str = Header(...), db: Session = Depends(...)):
    user = db.execute(select(User).where(User.openid == x_user_id)).scalar_one_or_none()
    if user is None:
        return success_response(_build_empty_user(x_user_id))  # 返回空用户，is_registered=false
    return success_response(_filter_avatar(user, db))  # 过滤未审核头像
```

最关键的改动：**不再通过 `get_or_create_user` 无条件创建空行**。只有注册时才会 INSERT 新行。

#### 5.5.2 审核记录插入时机

各类业务在创建/更新时，通过 `upload.py` 导出的 `create_image_review()` 函数自动创建审核记录：

```python
# routers/upload.py
def create_image_review(db, url, owner_type, owner_id=0, auto_approve=False):

    review = ImageReview(owner_type=owner_type, owner_id=owner_id, url=url, status=status)
```

### 7.3 请求封装

```javascript
// utils/request.js
const BASE_URL = 'http://192.168.43.34:8000/api'

function getUserId() {
  let id = wx.getStorageSync('openid')
  if (!id) {
    id = 'mp_' + Date.now() + '_' + Math.random().toString(36).slice(2)
    wx.setStorageSync('openid', id)
  }
  return id
}

function request(method, url, data) {
  return new Promise((resolve, reject) => {
    wx.request({
      url: BASE_URL + url,
      method,
      header: { 'X-User-Id': getUserId() },
      data,
      success(res) {
        if (res.data.code === 200) resolve(res.data)
        else reject({ code: res.data.code, message: res.data.message })
      },
      fail(err) { reject(err) }  // 网络错误
    })
  })
}
```

## 8. AI 交互过程实录

本章记录与 AI（TRAE）的交互过程中发现和解决的各类问题。

### 8.1 Token 安全加固

**问题**：Token 存储在 `localStorage`，关闭浏览器标签页不会清除，存在安全风险。

**交互**：用户提出 Token 安全问题 → AI 分析后建议改为 `sessionStorage` + 启动时主动验证 `/auth/me`。

**解决**：
1. `localStorage` → `sessionStorage`（关闭标签页自动清除）
2. `AuthProvider` 挂载时调用 `GET /api/auth/me` 验证 Token
3. 响应拦截器检测 401 → 清除 Token → 跳转登录页
4. `AuthGuard` 增加 `isValidating` 状态防止加载闪烁

### 8.2 Dashboard 数据全部为 0

**问题**：仪表盘显示咨询、电话、活动数量全为 0，但实际不为 0。

**交互**：用户报告数据展示问题 → AI 检查 Dashboard 页面 → 发现所有 `<Statistic value={0}>` 硬编码。

**解决**：
1. 后端新建 `GET /api/dashboard/stats` 统计 6 张表
2. 前端 `useEffect` 调接口 → `useState` 存结果
3. 页面改为 `value={stats.xxx_count}` 动态渲染

### 8.3 图片审核全链路缺陷

**问题**：上传图片没反应，审核表与业务表脱节。

**交互**：用户描述"上传图片没反应" → AI 全面审查 upload/business/news/event 上下游 → 发现 3 个问题。

**解决**：
1. **审核表是孤岛**：business/news/event 的 Create/Update 不创建审核记录，GET 也不过滤
   - 导出 `create_image_review()` 和 `is_image_approved()` 供各路由复用
2. **Business 前后端字段不一致**：前端传 `images: string[]`，后端期望 `logo: string`
   - 统一为 `logo: string` + 单图上传
3. **Event 前后端字段不一致**：前端 `location/start_time/end_time` vs 后端 `address/event_date/event_time`
   - 全部对齐后端 schema

### 8.4 小程序头像空白（三阶段排查）

#### 阶段一：审核表阻断

**问题**：`is_image_approved` 对用户上传头像（status=0）返回 false → `_filter_avatar` 将其设为空串。

**解决**：用户头像上传 `auto_approve=true`。

#### 阶段二：MIME 类型被拒绝

**问题**：微信 `wx.uploadFile` 的 Content-Type 不是标准 `image/jpeg` → 后端 400 拒绝 → 小程序 `fail()` 静默吞错。

**解决**：移除小程序上传端的 MIME 检查，Pillow 强制转 JPG。

#### 阶段三：upload fail 无提示

**问题**：`fail() { saveData(undefined) }` 静默丢弃，用户以为保存成功。

**解决**：`console.error` + `wx.showToast("头像上传失败")` 提示。

### 8.5 用户登录/注册数据覆盖 Bug

**问题**：登录已有账号显示"不存在"，注册已有账号覆盖旧数据。

**交互**：用户报告登录失败 + 数据异常 → AI 逐行审查 `user.py`。

**根因**：
1. 退出登录清空 `nickname=""` → 登录按昵称查找 → 找不到
2. 同设备注册新账号 → `get_or_create_user` 找到旧行 → 直接覆盖

**解决**：重写 register/login/logout 逻辑：
- 退出：`openid=""` 解绑，不删数据
- 注册：昵称+手机号唯一检查，旧 openid 解除绑定 → 新建行
- 登录：按手机号查找 → bcrypt 验证密码 → 切换 openid 绑定

### 8.6 500 Internal Server Error + 重复插入

**问题**：每次登录返回 500，users 表疯狂膨胀。

**根因**：`GET /api/users/me` 每次调用 `get_or_create_user` 无条件创建空行。

**解决**：
1. `GET /me` 只查不创
2. `_slot_` 占位符方案改为 `openid=""` 解绑（不会 unique 冲突）

### 8.7 管理员编辑密码不哈希

**问题**：管理员在后台编辑用户密码，存入明文，登录时 bcrypt 验证失败。

**解决**：后端 `update_user` 和 `update_self` 增加密码字段拦截 → `pwd_context.hash(password)` → `password_hash`。

### 8.8 审核管理页空列表

**问题**：图片审核列表永远是空的，审批/拒绝按钮无数据可操作。

**根因**：`res.data.items` → 实际数据在 `res.data.data.items`（差一层嵌套）。

**解决**：前端改用兼容取值 `body.data?.items ?? body.items ?? []`。
