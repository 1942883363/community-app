# 社区便民资讯平台 需求规格说明书

## 1. 项目概述

### 1.1 项目背景
为社区居民提供一个集中获取社区资讯、便民服务和反馈意见的一站式数字化平台，同时为社区管理者提供高效的信息发布和工单处理工具。

### 1.2 项目目标
- 微信小程序端：面向社区居民，提供资讯浏览、便民服务查询、意见反馈提交、活动报名等功能
- React 管理后台：面向社区管理员，提供资讯管理、分类管理、反馈处理、用户管理、图片审核等功能
- FastAPI 后端服务：提供统一的 RESTful API，管理所有业务数据

### 1.3 技术栈
| 层级 | 技术选型 | 版本 |
|------|----------|------|
| 后端框架 | FastAPI | 0.115+ |
| ORM | SQLAlchemy | 2.0+ |
| 数据库 | MySQL | 8.0 |
| 数据库驱动 | PyMySQL | 1.1+ |
| 认证 | python-jose + bcrypt + passlib | 3.3+ |
| 图片处理 | Pillow | 11+ |
| 管理前端 | React 18 + TypeScript + Ant Design 5 | 18 / 5 |
| 构建工具 | Vite | 5 |
| 微信小程序 | 原生框架 (WXML + WXSS + JS) | — |
| 外部 API | 高德地图公交 API | — |

## 2. 功能需求

### 2.1 管理员认证模块
| 编号 | 功能 | 描述 |
|------|------|------|
| AUTH-01 | 管理员登录 | 用户名+密码登录，bcrypt 验证，返回 JWT Token (HS256, 8h 过期) |
| AUTH-02 | Token 验证 | 所有管理端 API 需携带 `Authorization: Bearer <token>` |
| AUTH-03 | Token 存储 | sessionStorage 存储，关闭标签页自动清除 |
| AUTH-04 | Token 主动验证 | App 启动时调用 GET /api/auth/me，无效则清除 |

### 2.2 小程序用户系统
| 编号 | 功能 | 描述 |
|------|------|------|
| USER-01 | 设备标识 | 首次启动生成 UUID 存入 wx.Storage，后续请求通过 X-User-Id Header 传递 |
| USER-02 | 用户注册 | 昵称+手机号+密码注册，昵称唯一+手机号唯一校验，bcrypt 哈希密码 |
| USER-03 | 用户登录 | 手机号+密码登录，openid 绑定到目标用户行 |
| USER-04 | 退出登录 | openid 清空解绑，数据保留 |
| USER-05 | 个人信息 | GET/PUT /api/users/me 获取和修改个人资料（含密码修改） |
| USER-06 | GET /me 不创建空行 | 未注册用户返回 is_registered=false，不 INSERT |

### 2.3 资讯模块
| 编号 | 功能 | 描述 |
|------|------|------|
| NEWS-01 | 资讯分类管理 | 管理员可创建/编辑/删除资讯分类 |
| NEWS-02 | 资讯发布 | 管理员填写标题、内容、封面图、分类后发布 |
| NEWS-03 | 资讯列表 | 小程序端按分类筛选、分页加载、按发布时间倒序 |
| NEWS-04 | 资讯详情 | 小程序端查看资讯全文 |
| NEWS-05 | 资讯点赞 | 用户可点赞/取消点赞，记录用户点赞状态 |
| NEWS-06 | 浏览计数 | 每次查看详情，浏览次数 +1 |
| NEWS-07 | 置顶管理 | 管理员可置顶/取消置顶资讯 |
| NEWS-08 | 草稿/发布 | status 控制 0:草稿 1:已发布 |

### 2.4 意见反馈模块
| 编号 | 功能 | 描述 |
|------|------|------|
| FB-01 | 提交反馈 | 用户填写内容、联系方式（可选）、上传图片（可选） |
| FB-02 | 工单列表 | 管理员查看所有反馈，支持按状态筛选 |
| FB-03 | 工单处理 | 管理员可回复反馈、修改状态 |
| FB-04 | 状态管理 | 0:待处理 1:已处理 |

### 2.5 便民电话簿模块
| 编号 | 功能 | 描述 |
|------|------|------|
| PHONE-01 | 分类管理 | 管理员管理电话簿分类（名称+图标+排序） |
| PHONE-02 | 条目管理 | 管理员管理电话条目（名称、号码、排序） |
| PHONE-03 | 电话查询 | 小程序端按分类查询，支持一键拨打 |

### 2.6 社区活动报名模块
| 编号 | 功能 | 描述 |
|------|------|------|
| EVENT-01 | 活动管理 | 管理员发布/编辑/删除活动（标题、内容、封面、日期、时间、地点、名额） |

### 2.7 周边商家查询模块

### 2.8 公交地铁查询模块

| TRANSIT-02 | 线路到站 | 查询指定线路的实时到站时间 |

### 2.9 图片审核模块
| 编号 | 功能 | 描述 |
|------|------|------|
| REVIEW-01 | 审核记录创建 | 所有图片上传自动创建审核记录（管理后台操作 auto_approve=true） |
| REVIEW-02 | 审核列表 | 管理员查看所有审核记录，按状态筛选（待审核/已通过/已拒绝） |
| REVIEW-03 | 审核操作 | 管理员通过/拒绝图片 |
| REVIEW-04 | 图片过滤 | GET 端点根据审核状态过滤未通过图片（无记录默认放行） |

### 2.10 仪表盘统计
| 编号 | 功能 | 描述 |
|------|------|------|
| DASH-01 | 统计接口 | 一次请求统计 6 张表（news/feedback/phone/event/business/user）计数 |
| DASH-02 | 卡片展示 | 管理后台首页展示六维度统计卡片 |

## 3. 非功能需求

| 编号 | 类型 | 描述 |
|------|------|------|
| NF-01 | 响应格式 | 统一返回 `{"code": int, "message": str, "data": any}` |
| NF-02 | 分页 | 列表接口统一分页参数 `page`、`page_size`，返回 `{"items":[...],"total":N,"page":1,"page_size":20}` |
| NF-03 | CORS | 后端配置 CORS 中间件，允许所有来源跨域（开发阶段） |
| NF-04 | 文件上传 | 图片上传至本地 `uploads/images/` 目录，通过 `/uploads` 静态文件挂载访问 |
| NF-05 | 错误处理 | 全局异常捕获，返回统一错误格式 |
| NF-06 | 密码安全 | bcrypt 哈希存储，不定向还原 |
| NF-07 | 图片格式 | 小程序上传用 Pillow 强制转 JPG 统一输出 |
| NF-08 | MIME 兼容 | 小程序上传端不校验 MIME 类型（兼容 wx.uploadFile） |
