# 社区便民资讯平台 需求规格说明书

## 1. 项目概述

### 1.1 项目背景
为社区居民提供一个集中获取社区资讯、便民服务和反馈意见的一站式数字化平台，同时为社区管理者提供高效的信息发布和工单处理工具。

### 1.2 项目目标
- 微信小程序端：面向社区居民，提供资讯浏览、便民服务查询、意见反馈提交等功能
- React 管理后台：面向社区管理员，提供资讯管理、分类管理、反馈工单处理等功能
- FastAPI 后端服务：提供统一的 RESTful API，管理所有业务数据

### 1.3 技术栈
| 层级 | 技术选型 |
|------|----------|
| 后端框架 | FastAPI (Python 3.10+) |
| ORM | SQLAlchemy 2.0 |
| 数据库 | MySQL 8.0 |
| 认证 | JWT (python-jose) |
| 文件上传 | python-multipart |
| 管理前端 | React 18 + TypeScript + Ant Design 5 |
| 构建工具 | Vite |
| 微信小程序 | 原生框架 / Taro |
| 外部 API | 高德地图公交 API |

## 2. 功能需求

### 2.1 用户认证模块
| 编号 | 功能 | 描述 |
|------|------|------|
| AUTH-01 | 管理员登录 | 用户名+密码登录，返回 JWT Token |
| AUTH-02 | Token 验证 | 所有管理端 API 需携带 Bearer Token |
| AUTH-03 | 种子数据 | 系统初始化时创建默认管理员账号 |

### 2.2 资讯模块
| 编号 | 功能 | 描述 |
|------|------|------|
| NEWS-01 | 资讯分类管理 | 管理员可创建/编辑/删除资讯分类，支持两级层级 |
| NEWS-02 | 资讯发布 | 管理员填写标题、内容、封面图、分类、摘要后发布 |
| NEWS-03 | 资讯列表 | 小程序端按分类筛选、分页加载、按发布时间倒序 |
| NEWS-04 | 资讯详情 | 小程序端查看资讯全文（富文本） |
| NEWS-05 | 资讯点赞 | 用户可点赞/取消点赞，记录用户点赞状态 |
| NEWS-06 | 资讯分享 | 调用微信原生分享，传递标题和封面图 |
| NEWS-07 | 浏览计数 | 每次查看详情，浏览次数 +1 |

### 2.3 意见反馈模块
| 编号 | 功能 | 描述 |
|------|------|------|
| FB-01 | 提交反馈 | 用户填写内容、联系方式（可选）、上传图片（可选） |
| FB-02 | 工单列表 | 管理员查看所有反馈，支持按状态筛选 |
| FB-03 | 状态流转 | 待处理 → 处理中 → 已解决，记录操作人和时间 |
| FB-04 | 处理备注 | 管理员流转状态时可填写处理备注 |

### 2.4 便民电话簿模块
| 编号 | 功能 | 描述 |
|------|------|------|
| PHONE-01 | 分类管理 | 管理员管理电话簿分类（如物业维修、医疗服务等） |
| PHONE-02 | 条目管理 | 管理员管理电话条目（名称、号码、备注） |
| PHONE-03 | 电话查询 | 小程序端按分类查询，支持一键拨打 |

### 2.5 社区活动报名模块
| 编号 | 功能 | 描述 |
|------|------|------|
| EVENT-01 | 活动管理 | 管理员发布/编辑/删除活动（标题、时间、地点、名额、详情） |
| EVENT-02 | 活动列表 | 小程序端查看活动列表和详情 |
| EVENT-03 | 在线报名 | 用户填写姓名和电话报名，名额减1 |
| EVENT-04 | 名额校验 | 名额满时拒绝报名并提示 |

### 2.6 周边商家查询模块
| 编号 | 功能 | 描述 |
|------|------|------|
| BIZ-01 | 商家分类 | 管理员管理商家分类（餐饮、超市、药店等） |
| BIZ-02 | 商家管理 | 管理员管理商家信息（名称、地址、电话、简介、图片） |
| BIZ-03 | 商家查询 | 小程序端按分类查询商家列表和详情 |

### 2.7 公交地铁查询模块
| 编号 | 功能 | 描述 |
|------|------|------|
| TRANSIT-01 | 周边站点 | 根据经纬度查询周边公交/地铁站点 |
| TRANSIT-02 | 线路到站 | 查询指定线路的实时到站时间 |
| TRANSIT-03 | Skill 封装 | 封装为 TRAE Skill，方便复用和扩展 |

## 3. 非功能需求

| 编号 | 类型 | 描述 |
|------|------|------|
| NF-01 | 响应格式 | 统一返回 `{"code": int, "message": str, "data": any}` |
| NF-02 | 分页 | 列表接口统一分页参数 `page`、`page_size`，返回 `total`、`items` |
| NF-03 | CORS | 后端配置 CORS 中间件，允许管理前端跨域访问 |
| NF-04 | 文件上传 | 图片上传至本地 `uploads/` 目录，通过静态文件服务访问 |
| NF-05 | 错误处理 | 全局异常捕获，返回统一错误格式 |
